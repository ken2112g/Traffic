# Product

## Register

product

## Users

Solo operator managing a multi-platform social media engagement engine from a VPS. Opens the dashboard to check what's running, catch failures fast, and trigger campaigns — not to learn the tool, but to control it. Always in operator mode: focused, task-driven, expects the UI to keep up with them.

## Product Purpose

A web dashboard for the "kéo traffic" automation engine. The engine manages 100+ social media sub-accounts (Pinterest, Instagram, TikTok) that automatically like, follow, comment, and view content on target accounts for POD SEO. The dashboard exposes: real-time task logs, engine health stats, account management (add / ban / delete), campaign management (create / trigger / toggle), and proxy health. Success = operator can see the system state at a glance and act on anomalies within seconds.

## Brand Personality

Precise, operational, unobtrusive. The tool runs in the background doing serious work; the UI should feel the same way — no decoration that isn't earning its place, no color that isn't signaling status. Think instrument panel, not landing page.

## Anti-references

- Generic SaaS dashboards with hero metrics, gradient cards, and big marketing numbers
- Cream / warm-neutral backgrounds — this is a dark ops tool, not a productivity app
- Pastel status colors that don't differentiate clearly between ok / warn / error
- Onboarding flows, empty-state illustrations, or any element that assumes the user is new

## Design Principles

1. **Status at a glance** — Running, idle, banned, failed must read instantly without reading labels. Color and shape carry status; text confirms it.
2. **Density without clutter** — Pack information tightly, but let each piece breathe just enough. No wasted whitespace, no decorative chrome.
3. **One click to act** — Trigger a campaign, ban an account, test a proxy. Common operator actions are always visible, never buried in submenus.
4. **Live by default** — Logs and stats update automatically. The dashboard should feel like a live instrument, not a report.
5. **Trust the operator** — No excessive confirmation dialogs, no dumbed-down copy. The user knows what they're doing.

## Accessibility & Inclusion

WCAG AA contrast minimum. No motion-only status indicators (always pair with color + shape). Keyboard navigable tables. No requirements beyond standard; single-user tool.
